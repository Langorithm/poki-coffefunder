## Notes

- Confirmed Day 1 pending steps were all done: EventBus autoload registered, `local_player_input.gd` updated, `AimJoystick` script swapped in `world.tscn`. Movement is working end to end.
- Caught a memory hygiene issue: absence of a note in memory ≠ feature not built. Going forward, session-end state must include a **Verified Working** section. Task states are the authoritative source for what's done.
- Input design is Brawl Stars: left stick moves, right stick drag-aim-fires, tap auto-aims. Mouse players have precise aim and need no help. Physical controller gets a dedicated button for auto-aim (TBD). Haptics worth investigating for virtual joystick feedback — deferred.
- `special` is the B-button power move (dash placeholder), not a second attack. Don't overload it.
- Auto-aim fires at current position, not predicted — moving sideways dodges it. Skill expression is learning to lead targets with drag-aim. Encounter design should stress-test this.
- Mouse shooting was broken by full-screen background `ColorRect` with default `MOUSE_FILTER_STOP`. Fixed with `mouse_filter = 2`.
- Ammo/reload signals live on Player directly — EventBus would require player identification for MP, not worth it at this scale.
- EventBus camera_shake not MP-compatible (all cameras fire). Noted in GameCamera docstring, acceptable for now.
- `AimPivot` pattern: scene sets shape/offset of AimLine child visually; code only drives parent position + rotation.
- Ammo pip tracking: `_pips_offset` read from scene in `_ready`, added to player screen pos each frame. Editor sets the offset.
