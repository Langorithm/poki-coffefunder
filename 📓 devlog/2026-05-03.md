# 2026-05-03 — Day 1

Sprint kickoff. Half-day session. Architecture-first: no feature work until the foundation is solid.

## Goals

### Project scaffold
- [x] Create `src/` folder structure per Godot best practices #TODO
- [x] Define all Input Map actions in `project.godot` (move, aim, shoot, special)

### Player architecture
- [x] `PlayerInput` abstract base class
	-  defines the interface Player depends on
- [x] `LocalPlayerInput`
	-  reads Input Map (WASD + gamepad) and virtual joystick signals; passes `motion_vector`, `aim_vector`, and action signals to Player
- [x] `Player` (CharacterBody2D) — movement only; receives vectors from `LocalPlayerInput`, never reads `Input` directly

> [!NOTE] Naming decision
> `LocalPlayerInput` / `NetworkPlayerInput` (not `PlayerController`) — chosen for clarity and to avoid confusion with `CharacterBody2D`. See [[adr-001-input-decoupling]].

### World & HUD
- [x] `world.tscn` — arena floor, camera, Player instance; enough to walk around and test
- [x] HUD scene stub — `CanvasLayer` with virtual joystick addon placed; hidden on desktop via `OS.has_feature("mobile")`

### Export size
- [x] Research validated — see [[export-size-optimization]]
	- Poki CDN compresses server-side; 8–10MB target is compressed download size. Baseline: 36MB uncompressed / 8.9MB gzip (no assets). Already within range.
- [x] Set up Build Configuration Profile
	- Saved as `initial_trivial.gdbuild` at project root. Labeled trivial — autodetect too aggressive this early, revisit at feature-complete.
- Custom export templates deferred to pre-submission

### Closing thoughts

Did a full project setup. Godot code itself is pretty sparse but I feel like we've accelerated future work quite substantially, both technically and creatively.

The Obsidian+Claude setup is quite amazing and I love having the ability to store all the insights and brainstorming from my conversations with the LLM in a wiki style, I love that it can read and write as needed. It's quite a game changer.

The Poki size [[export-size-optimization#Export Size Optimization|export size optimization]] findings were remarkable. It's something that had been eating at me since my last web project which I uploaded to itch. Itch does not compress so I could tell downloads were slow, and I knew Poki needs fast serving. Turns out we're already within their target range with zero assets.

I like working like this.
